## Step 8 ‚Äî API Endpoints

| #   | Endpoint                                      | Method | Description                                           | File                                   |
| --- | --------------------------------------------- | ------ | ----------------------------------------------------- | -------------------------------------- |
| 8.1 | `GET /api/workout-sessions/active`            | GET    | Returns active session or 404                         | `Endpoints/WorkoutSessionEndpoints.cs` |
| 8.2 | `GET /api/exercises`                          | GET    | Returns seeded, read-only exercise list               | `Endpoints/ExerciseEndpoints.cs`       |
| 8.3 | `POST /api/workout-sessions/{sessionId}/sets` | POST   | Logs a new set with validation ‚Üí 201 or 400           | `Endpoints/WorkoutSetEndpoints.cs`     |
| 8.4 | `GET /api/workout-sessions/{sessionId}/sets`  | GET    | Returns all sets in session, ordered by CreatedAt asc | `Endpoints/WorkoutSetEndpoints.cs`     |

### POST Validation ‚Äî custom validation behavior

- WeightValue: decimal > 0 and ‚â§ 500
- Reps: int 1‚Äì100
- Rir: byte 0‚Äì10
- ExerciseId: must exist in database
- Response 400: `{ errors: [{ field, message }] }`
